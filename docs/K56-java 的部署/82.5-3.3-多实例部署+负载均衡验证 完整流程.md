# 多实例部署 + 负载均衡验证 完整流程

## 流程概览

```
mvn package 打包成 jar
    ↓
在 Controller 里加日志，确认请求被哪个实例处理
    ↓
用不同端口启动多个 jar 实例（8080/8081/8082）
    ↓
Nginx 负载均衡分发请求
    ↓
观察多个实例的日志，验证轮询效果
```

---

## 第一步：打包项目为 jar

```bash
mvn package
```

执行成功后输出 `BUILD SUCCESS`，jar 文件生成在 `target/` 目录下，例如：

```
target/xdml-v1-0.1.0.jar
```

**作用：** 把项目编译并打包成一个可独立运行的 fat jar（包含所有依赖），之后可以直接用 `java -jar` 启动，不依赖 IDE。

---

## 第二步：在 Controller 里加日志

在 `IndexController.java` 的请求处理方法里加一行打印：

```java
@GetMapping("/")
public ModelAndView index() {
    Map<String, Object> map = new HashMap<>();
    map.put("aaa", "111");

    final List<Match> matches = matchService.getMatches();
    map.put("matches", matches);

    System.out.println("Get request!");   // ← 加这行

    return new ModelAndView(viewName: "index", map);
}
```

**作用：** 每次有请求进来，控制台就会打印 `Get request!`。启动多个实例后，通过观察哪个终端窗口打印了这行日志，就能直观看出 Nginx 把请求分发给了哪个实例，从而验证负载均衡是否生效。

> 加完日志后需要重新执行 `mvn package` 生成新的 jar。

---

## 第三步：用不同端口启动多个实例

打开终端窗口，执行：

```bash
# 实例 2，指定 8081 端口
java -Dserver.port=8081 -jar target/xdml-v1-0.1.0.jar

```

**参数说明：**
- `-Dserver.port=8081`：通过 JVM 系统属性覆盖 `application.properties` 里的端口配置，无需修改配置文件

**作用：** 模拟生产环境中多台服务器同时运行同一个应用的场景。三个实例监听不同端口，Nginx upstream 里配置了这三个端口，请求会被轮询分发。

> 如果启动时报 `port 8080 is already in use`，说明该端口被占用，用 `lsof -i:8080` 查找并关闭占用进程。

---

## 第四步：验证负载均衡效果

在浏览器多次访问：

```
http://abc.com
```

或用 curl 连续发多次请求：

```bash
curl http://abc.com
curl http://abc.com
curl http://abc.com
```

**观察三个终端窗口**，可以看到 `Get request!` 依次在不同实例的终端里打印：

```
# 实例 1（8080）终端
Get request!

# 实例 2（8081）终端
Get request!

# 实例 3（8082）终端
Get request!
```

Nginx 日志（`172.17.0.1` 是 Docker 网关 IP）也可以看到请求记录：

```
172.17.0.1 - - [09/Dec/2019:14:33:08 +0000] "GET / HTTP/1.1" 200 719 ...
172.17.0.1 - - [09/Dec/2019:14:33:14 +0000] "GET / HTTP/1.1" 200 719 ...
```

**作用：** 验证 Nginx 的轮询负载均衡策略确实在工作，请求被均匀分发到各个实例，而不是全部打到同一台。

---

## 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `port 8080 is already in use` | 端口被占用 | `lsof -i:8080` 查找进程，`kill <PID>` 关闭 |
| 多次访问只有一个实例打印日志 | Nginx 配置未生效或浏览器有缓存 | `docker restart my-nginx`，用 `curl` 代替浏览器测试 |
| `java -jar` 找不到 jar 文件 | 未执行 `mvn package` 或路径不对 | 先执行 `mvn package`，再确认 `target/` 下有 jar 文件 |
| 加了日志但没有打印 | jar 是旧版本，没有重新打包 | 修改代码后重新执行 `mvn package` |
