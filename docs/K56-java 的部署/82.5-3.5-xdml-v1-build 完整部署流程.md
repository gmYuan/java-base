# xdml-v1-build 完整部署流程

## 总体流程概览

```
Docker 启动 MySQL + Redis
    ↓
创建数据库，配置项目依赖和连接信息
    ↓
执行 Flyway 建表
    ↓
本地启动项目验证功能正常
    ↓
修改 hosts，配置本地域名映射
    ↓
查本机 IP，编写 nginx.conf，配置负载均衡
    ↓
Docker 启动 Nginx
    ↓
在 Controller 加日志，打包 jar
    ↓
编写 Dockerfile，构建应用镜像
    ↓
用外部配置挂载启动容器
    ↓
验证容器内应用正常 + 验证负载均衡效果
```

---

## 第一部分：基础服务启动

### 1.1 启动 MySQL

```bash
docker run -d --name my-mysql -e MYSQL_ROOT_PASSWORD=root -p 3306:3306 mysql
```

**参数说明：**
- `-d`：后台运行容器
- `--name my-mysql`：给容器起名，之后可用 `docker start my-mysql`、`docker stop my-mysql` 等命令操作
- `-e MYSQL_ROOT_PASSWORD=root`：设置 root 用户密码
- `-p 3306:3306`：把容器的 3306 端口映射到本机 3306 端口

**作用：** MySQL 不需要本地安装，直接用 Docker 拉起一个容器实例，Java 项目通过 `localhost:3306` 连接，和本地安装的 MySQL 完全一样。

### 1.2 启动 Redis

```bash
docker run -d --name my-redis -p 6379:6379 redis:6.2
```

**验证 Redis 是否正常：**

```bash
docker exec -it my-redis redis-cli ping
# 返回 PONG 即成功
```

> **重启 Mac 后**，容器会停止，需手动重启：
> ```bash
> docker start my-mysql my-redis
> ```

## 第二部分：数据库初始化

### 2.1 在 IDEA 连接 MySQL，创建数据库

在 IDEA 右侧 **Database 面板** 新建连接：
- Driver: `MySQL`，Host: `localhost`，Port: `3306`
- User: `root`，Password: `root`
- 点 **Test Connection** 验证连通性，再点 **OK**

在 SQL Console 里执行：

```sql
create database `match`;
```

> `match` 是 MySQL 保留关键字，必须用反引号包裹，否则报语法错误。

**作用：** MySQL 启动后默认没有业务数据库，需要手动创建。Flyway 后续会在这个数据库里建表。

### 2.2 pom.xml 加入 MySQL 驱动依赖

```xml
<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-java</artifactId>
    <version>8.0.18</version>
</dependency>
```

**作用：** Java 连接 MySQL 必须有 JDBC 驱动。`driver-class-name=com.mysql.cj.jdbc.Driver` 这个类就来自这个 jar 包，没有它项目启动时会报 `ClassNotFoundException`。

### 2.3 pom.xml 配置 flyway-maven-plugin

```xml
<plugin>
    <groupId>org.flywaydb</groupId>
    <artifactId>flyway-maven-plugin</artifactId>
    <version>6.0.7</version>
    <configuration>
        <url>jdbc:mysql://localhost:3306/match</url>
        <user>root</user>
        <password>root</password>
    </configuration>
</plugin>
```

**作用：** 让 Maven 的 Flyway 插件知道要连接哪个数据库来执行迁移脚本。

### 2.4 执行 Flyway 建表

```bash
# 第一次执行
mvn flyway:migrate

# 如果之前已执行过迁移，先清空再重建
mvn flyway:clean flyway:migrate
```

Flyway 会自动扫描并执行 `src/main/resources/db/migration/V1__CreateTables.sql`：

```sql
create table `USER`(
    id   bigint primary key auto_increment,
    name varchar(100)
);

create table `MATCH`(
    id      bigint primary key auto_increment,
    user_id bigint,
    score int
);

insert into `USER`(id, name) values (1, 'AAA');
insert into `USER`(id, name) values (2, 'BBB');
insert into `USER`(id, name) values (3, 'CCC');

insert into `MATCH`(id, user_id, score) values (1, 1, 1000);
insert into `MATCH`(id, user_id, score) values (2, 1, 2000);
insert into `MATCH`(id, user_id, score) values (3, 2, 500);
insert into `MATCH`(id, user_id, score) values (4, 3, 300);
```

> **注意：** MySQL 不支持双引号标识符（`"USER"`），必须用反引号（`` `USER` ``）。H2 用双引号，切换到 MySQL 时需修改。

**作用：**
- 不需要手动在数据库里建表
- SQL 文件纳入版本控制，团队协作时执行同一套脚本，数据库结构保持一致
- `flyway:clean` 会清空数据库重建，`flyway:migrate` 只执行新增的迁移文件

## 第三部分：MyBatis 说明

MyBatis 是 ORM 框架，负责把 SQL 查询结果自动映射成 Java 对象，避免手动解析 ResultSet。

项目中的 Mapper 示例（`src/main/resources/db/mybatis/` 目录下）：

```xml
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
    "https://mybatis.org/dtd/mybatis-3-mapper.dtd">

<mapper namespace="MyMapper">
    <select id="getMatches" resultMap="match">
        select user.id as user_id,
               user.name as user_name,
               t.score as score
        from `USER`
        join (
            select user_id, sum(score) as score
            from `MATCH`
            group by user_id
        ) t on `USER`.id = t.user_id
    </select>

    <resultMap id="match" type="xdml.Match">
        <result property="score" column="score"/>
        <association property="user" javaType="xdml.User">
            <result property="id" column="user_id"/>
            <result property="name" column="user_name"/>
        </association>
    </resultMap>
</mapper>
```

MyBatis 配置文件路径在 `application.properties` 中指定：

```properties
mybatis.config-location=classpath:db/mybatis/config.xml
```

## 第五部分：配置 Nginx 负载均衡

### 5.1 修改 hosts 文件，配置本地域名

```bash
sudo vim /etc/hosts
```

在文件末尾添加：

```
127.0.0.1    abc.com
```

**作用：** `/etc/hosts` 是操作系统本地的 DNS 解析文件。把 `abc.com` 映射到 `127.0.0.1`（本机），浏览器访问 `abc.com` 时请求会打到本机 Nginx，而不是去真实互联网查找这个域名。

### 5.2 查询本机 IP

```bash
ifconfig
# 找 en0 下的 inet，例如：192.168.31.250
```

**作用：** Docker 容器内部无法用 `localhost` 访问宿主机的服务（容器有自己独立的网络命名空间），必须用宿主机的局域网 IP。

### 5.3 查看 80 端口是否被占用

```bash
lsof -i:80
```

**作用：** 确认 80 端口没有被其他进程占用（如 Apache），避免 Nginx 启动时端口冲突。

### 5.4 编写 nginx.conf

在项目目录下创建 `nginx.conf`（与 `pom.xml` 同级）：

```bash
vi nginx.conf
```

配置内容：

```nginx
events { }

http {
    upstream myapp1 {
        server 192.168.31.250:8080;
        server 192.168.31.250:8081;
        server 192.168.31.250:8082;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://myapp1;
        }
    }
}
```

> **为什么要有 `events { }`：** 这是 Nginx 配置文件的必填块，即使内容为空也必须写。Nginx 启动时必须解析到这个块，否则报错 `no "events" section in configuration`。留空表示全部使用默认配置。

**作用：**
- `upstream myapp1`：定义后端服务器组，包含 3 个应用实例（8080/8081/8082 端口）
- `listen 80`：Nginx 监听 80 端口（HTTP 默认端口）
- `proxy_pass http://myapp1`：把所有请求转发到 upstream，默认使用**轮询**策略

> 注意：upstream 里用的是宿主机局域网 IP，不能用 `localhost`，因为这个配置是在 Docker 容器内执行的。

### 5.5 删除旧的 Nginx 容器（如有）

```bash
docker ps
docker rm -f <旧容器ID>
```

**作用：** 如果之前启动过 Nginx 容器，需要先删除，否则端口冲突或配置不会更新。

### 5.6 Docker 启动 Nginx

```bash
docker run --restart=always --name my-nginx \
  -v `pwd`/nginx.conf:/etc/nginx/nginx.conf:ro \
  -p 80:80 -d nginx
```

**参数说明：**
- `--restart=always`：Docker 重启后自动拉起容器
- `-v \`pwd\`/nginx.conf:/etc/nginx/nginx.conf:ro`：把当前目录的 `nginx.conf` 挂载到容器内，`:ro` 表示只读
- `-p 80:80`：把容器的 80 端口映射到本机 80 端口
- `-d`：后台运行

**作用：** Nginx 不需要手动安装，用 Docker 拉起容器，配置文件通过 `-v` 挂载进去。

### 5.7 修改配置后重新加载

```bash
# 方式一：重启容器
docker restart my-nginx

# 方式二：向容器内 Nginx 发送 reload 信号（不中断服务）
docker exec -it my-nginx nginx -s reload
```

**作用：** Nginx 启动后配置不会自动热更新，需要手动触发 reload 才能让新配置生效。

## 第六部分：打包应用为 Docker 镜像

### 6.1 在 Controller 加请求日志（用于验证负载均衡）

在 `IndexController.java` 的请求处理方法里加一行打印：

```java
@GetMapping("/")
public ModelAndView index() {
    Map<String, Object> map = new HashMap<>();
    map.put("aaa", "111");

    final List<Match> matches = matchService.getMatches();
    map.put("matches", matches);

    System.out.println("Get request!");   // ← 加这行

    return new ModelAndView("index", map);
}
```

**作用：** 每次有请求进来，控制台就会打印 `Get request!`。启动多个实例后，通过观察哪个终端或容器打印了这行日志，就能直观看出 Nginx 把请求分发给了哪个实例。

> 加完日志后需要重新执行 `mvn package` 生成新的 jar。

### 6.2 打包 jar

```bash
mvn package -DskipTests
```

成功后生成 `target/xdml-v1-0.1.0.jar`。

确认 MyBatis 配置文件在 jar 包内的实际路径（外部配置文件名必须与之一致）：

```bash
jar tf target/xdml-v1-0.1.0.jar | grep mybatis
# 输出：BOOT-INF/classes/db/mybatis/config.xml
```

### 6.3 编写 Dockerfile

在项目根目录创建 `Dockerfile`（与 `pom.xml` 同级）：

```dockerfile
FROM amazoncorretto:8-alpine

RUN mkdir /app

WORKDIR /app

COPY target/xdml-v1-0.1.0.jar /app

EXPOSE 8080

CMD [ "java", "-jar", "xdml-v1-0.1.0.jar" ]
```

| 指令 | 作用 |
|------|------|
| `FROM amazoncorretto:8-alpine` | 基础镜像，JDK 8 + Alpine 精简 Linux，原生支持 arm64（Apple Silicon） |
| `RUN mkdir /app` | 容器内创建 `/app` 目录 |
| `WORKDIR /app` | 设置工作目录，后续命令都在 `/app` 下执行 |
| `COPY target/xdml-v1-0.1.0.jar /app` | 把本机打好的 jar 复制进镜像 |
| `EXPOSE 8080` | 声明容器对外暴露 8080 端口（仅文档作用，实际映射需 `-p`） |
| `CMD [...]` | 容器启动时执行的命令，即 `java -jar xdml-v1-0.1.0.jar` |

> **Apple Silicon 注意：** `java:openjdk-8u111-alpine` 是 x86 架构，在 M 系列 Mac 上容器会立刻退出且日志为空。必须改用 `amazoncorretto:8-alpine`（原生支持 arm64）。

### 6.4 构建镜像

```bash
docker build -t xdml-v1-build:0.1.0 .
```

构建完成后查看镜像：

```bash
docker images
# REPOSITORY       TAG     IMAGE ID
# xdml-v1-build    0.1.0   xxxxx
```

如果构建时没有指定 `-t`，镜像 REPOSITORY 和 TAG 都是 `<none>`，用 `docker tag` 给它命名：

```bash
docker tag <IMAGE_ID> xdml-v1-build:0.1.0
```

如果需要重命名已有镜像，先打新 tag 再删旧 tag：

```bash
docker tag xdml-v1:0.1.0 xdml-v1-build:0.1.0
docker rmi xdml-v1:0.1.0
```

## 第七部分：用外部配置启动容器

### 7.1 外部配置文件（项目根目录 application.properties）

容器内不能用 `localhost`，需把宿主机 IP 写入外部配置：

```properties
spring.datasource.url=jdbc:mysql://192.168.31.250:3306/match
spring.datasource.username=root
spring.datasource.password=root
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
mybatis.config-location=classpath:db/mybatis/config.xml
spring.aop.proxy-target-class=true

spring.redis.host=192.168.31.250
spring.redis.port=6379
```

> **为什么不用 localhost：** 容器有独立的网络命名空间，`localhost` 指的是容器自身，无法访问宿主机上的 MySQL 和 Redis。必须改成宿主机的局域网 IP。
>
> **mybatis.config-location 文件名** 必须和 jar 包内一致（是 `config.xml` 不是 `mybatis-config.xml`），可用 `jar tf target/*.jar | grep mybatis` 确认。

### 7.2 Spring Boot 配置文件加载优先级

| 优先级 | 位置 |
|--------|------|
| 1（最高） | 当前工作目录的 `/config` 子目录 |
| 2 | 当前工作目录 |
| 3 | classpath 下的 `/config` 包 |
| 4（最低） | classpath 根目录（jar 包内的 `application.properties`） |

容器的工作目录是 `/app`（`WORKDIR /app`），把配置挂载到 `/app/config/application.properties`，Spring Boot 会优先读取这份外部配置，**覆盖** jar 包内打包的配置文件。

**作用：** 不需要重新构建镜像就能修改配置，修改配置只需改文件再重启容器即可。

### 7.3 启动容器

```bash
docker run -d --name my-xdml-build-dock \
  -v `pwd`/application.properties:/app/config/application.properties \
  -p 8082:8080 xdml-v1-build:0.1.0
```

**参数说明：**
- `-v \`pwd\`/application.properties:/app/config/application.properties`：把当前目录的 `application.properties` 挂载到容器内的 `/app/config/` 目录
- `-p 8082:8080`：把容器内的 8080 端口映射到宿主机的 8082 端口
- `--name my-xdml-build-dock`：给容器命名，便于后续管理

也可以先启动再给容器改名：

```bash
docker rename <原容器名> my-xdml-build-dock
```

### 7.4 查看实时日志

```bash
docker logs -f my-xdml-build-dock
```

`-f` 表示持续跟踪日志（follow）。刷新浏览器页面，终端里每次都会打印：

```
Get request!
```

看到这行日志说明请求成功进入了容器内的应用。

## 第八部分：本地多实例 + 验证负载均衡

在 Docker 容器方案之外，也可以直接用 `java -jar` 在本地同时启动多个实例：

```bash
# 实例 1（默认 8080，直接启动）
java -jar target/xdml-v1-0.1.0.jar

# 实例 2（指定 8081 端口）
java -Dserver.port=8081 -jar target/xdml-v1-0.1.0.jar

# 实例 3（指定 8082 端口）
java -Dserver.port=8082 -jar target/xdml-v1-0.1.0.jar
```

**参数说明：** `-Dserver.port=8081` 通过 JVM 系统属性覆盖 `application.properties` 里的端口配置，无需修改配置文件。

> 如果报 `port 8080 is already in use`，用 `lsof -i:8080` 找到并 `kill <PID>` 关闭占用进程。

### 验证负载均衡效果

多次访问 `http://abc.com`，或用 curl 连续发请求：

```bash
curl http://abc.com
curl http://abc.com
curl http://abc.com
```

**观察三个终端窗口**，可以看到 `Get request!` 依次在不同实例的终端里打印：

```
# 实例 1（8080）终端
Get request!

# 实例 2（8081）终端
Get request!

# 实例 3（8082）终端
Get request!
```

也可以进入 Nginx 容器内测试网络连通性：

```bash
docker exec -it my-nginx bash
# 容器内执行
curl http://192.168.31.250:8080
```

> 注意：Nginx 的基础镜像里没有安装 `ping` 命令，用 `curl` 代替测试连通性。

**Nginx 日志**（`172.17.0.1` 是 Docker 网关 IP）也可以看到请求记录：

```
172.17.0.1 - - [09/Dec/2019:14:33:08 +0000] "GET / HTTP/1.1" 200 719 ...
172.17.0.1 - - [09/Dec/2019:14:33:14 +0000] "GET / HTTP/1.1" 200 719 ...
```

**作用：** 验证 Nginx 的轮询负载均衡策略确实在工作，请求被均匀分发到各个实例，而不是全部打到同一台。

## 常见问题汇总

| 问题 | 原因 | 解决 |
|------|------|------|
| `create database match` 报语法错误 | `match` 是 MySQL 关键字 | 改成 `` create database `match` `` |
| Flyway SQL 报语法错误 | SQL 文件用了双引号（H2 语法） | 改成反引号（MySQL 语法） |
| `ClassNotFoundException: com.mysql.cj.jdbc.Driver` | 缺少 MySQL 驱动依赖 | pom.xml 加入 `mysql-connector-java` |
| Flyway 报表已存在 | 之前已执行过迁移 | 先执行 `mvn flyway:clean` 再 `flyway:migrate` |
| `mvn` 命令找不到 | Maven 未安装或未配置 PATH | 用绝对路径 `/Users/ygm/Library/apache-maven-3.9.14/bin/mvn` |
| Docker 镜像下载中断 | 镜像源不稳定 | 换镜像源：`docker.m.daocloud.io`，重试 `docker pull` |
| `docker build` 报找不到 jar | 未打包或路径不对 | 先执行 `mvn package`，确认 `target/` 下有 jar |
| 容器启动后立刻退出，日志为空 | 基础镜像是 x86，M 系列 Mac 不兼容 | Dockerfile 改用 `amazoncorretto:8-alpine` 重新构建 |
| 容器连不上 MySQL/Redis | 配置用了 `localhost` | 外部配置改成宿主机局域网 IP（`ifconfig` 查） |
| `FileNotFoundException: db/mybatis/config.xml` | 外部配置文件名与 jar 包内不一致 | `jar tf target/*.jar \| grep mybatis` 确认实际文件名 |
| 修改了配置但没生效 | 容器未重启 | `docker restart <容器名>` |
| 外部配置未覆盖 jar 内配置 | 挂载路径不对 | 确认挂载到 `WORKDIR/config/application.properties` |
| 端口已被占用 | 已有容器或进程占用该端口 | `docker ps` 查容器，`lsof -i:<端口>` 查进程，`kill <PID>` 关闭 |
| 多次访问只有一个实例打印日志 | Nginx 配置未生效或浏览器有缓存 | `docker restart my-nginx`，用 `curl` 代替浏览器测试 |
| Nginx 改配置后不生效 | 未重新加载 | `docker restart my-nginx` 或 `nginx -s reload` |
| 容器内 `ping abc.com` 报 `command not found` | Nginx 镜像没有内置 `ping` | 改用 `curl` 测试连通性 |
| hosts 改后浏览器仍访问公网 | 浏览器 DNS 缓存 | 用 `curl abc.com` 测试，或清除浏览器缓存 |
