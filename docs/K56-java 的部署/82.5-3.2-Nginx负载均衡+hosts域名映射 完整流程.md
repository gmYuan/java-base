# Nginx 负载均衡 + hosts 域名映射 完整流程

## 流程概览

```
修改 hosts 文件，配置本地域名映射
    ↓
确认应用服务器运行在多个端口（8080/8081/8082）
    ↓
编写 nginx.conf，配置 upstream 负载均衡
    ↓
用 Docker 启动 Nginx 容器，挂载配置文件
    ↓
验证：通过域名访问，Nginx 自动分发请求
```

---

## 第一步：修改 hosts 文件，配置本地域名映射

```bash
sudo vim /etc/hosts
```

在文件末尾添加：

```
127.0.0.1    abc.com
```

**作用：** `/etc/hosts` 是操作系统本地的 DNS 解析文件。把 `abc.com` 映射到 `127.0.0.1`（本机），这样在浏览器访问 `abc.com` 时，请求会发到本机的 Nginx，而不是去真实互联网查找这个域名。

> 注意：`www.douban.com` 那行前面有 `#` 是注释掉的，不会生效。

---

## 第二步：查看本机 IP 地址

```bash
ifconfig
```

在输出里找 `en0` 下面的 `inet` 行，例如：

```
inet  192.168.31.250 netmask 0xffffff00 broadcast 192.168.0.255
```

记下这个 IP（`192.168.31.250`），后面 nginx.conf 里需要用到。

**作用：** Docker 容器内部无法用 `localhost` 访问宿主机的服务（因为容器有自己独立的网络），必须用宿主机的局域网 IP。

---

## 第三步：查看 80 端口占用情况

```bash
lsof -i:80
```

**作用：** 确认 80 端口没有被其他进程占用（如 Apache），避免 Nginx 启动时端口冲突。

---

## 第四步：编写 nginx.conf

```bash
vi nginx.conf
```

配置内容：

> **为什么要有 `events { }`：** 这是 Nginx 配置文件的必填块，即使内容为空也必须写。Nginx 启动时必须解析到这个块，否则报错 `no "events" section in configuration`。留空表示全部使用默认配置（默认连接数、默认 IO 模型等）。

```nginx
events { }
http {
    upstream myapp1 {
        server  192.168.31.250:8080;
        server  192.168.31.250:8081;
        server  192.168.31.250:8082;
    }

    server {
        listen 80;

        location / {
            proxy_pass http://myapp1;
        }
    }
}
```

**作用：**
- `upstream myapp1`：定义一个后端服务器组，包含 3 个应用实例（8080/8081/8082 端口）
- `listen 80`：Nginx 监听 80 端口（HTTP 默认端口）
- `proxy_pass http://myapp1`：把所有请求转发到 upstream 里的服务器，默认使用**轮询**策略，依次分发给三台服务器

> 注意：upstream 里用的是宿主机局域网 IP（` 192.168.31.250`），不能用 `localhost`，因为这个配置是在 Docker 容器内执行的。

---

## 第五步：删除旧的 Nginx 容器（如有）

```bash
docker ps
docker rm -f <旧容器ID>
```

**作用：** 如果之前启动过 Nginx 容器，需要先删除，否则端口冲突或配置不会更新。

---

## 第六步：用 Docker 启动 Nginx，挂载配置文件

```bash
docker run --restart=always --name my-nginx -v `pwd`/nginx.conf:/etc/nginx/nginx.conf:ro -p 80:80 -d nginx
```

**参数说明：**
- `--restart=always`：Docker 重启后自动拉起容器
- `-v \`pwd\`/nginx.conf:/etc/nginx/nginx.conf:ro`：把当前目录的 `nginx.conf` 挂载到容器内，`:ro` 表示只读
- `-p 80:80`：把容器的 80 端口映射到本机 80 端口
- `-d`：后台运行

**作用：** Nginx 不需要手动安装，用 Docker 拉起一个容器，配置文件通过 `-v` 挂载进去，容器内的 Nginx 会读取这份配置启动。

---

## 第七步：验证 Nginx 容器是否读取到了最新配置

修改配置后，需要让 Nginx 重新加载：

```bash
# 方式一：重启容器
docker restart <容器ID>

# 方式二：向容器内 Nginx 发送 reload 信号（不中断服务）
docker exec -it <容器ID> nginx -s reload
```

**作用：** Nginx 启动后配置不会自动热更新，需要手动触发 reload 才能让新配置生效。

---

## 第八步：验证负载均衡效果

在浏览器访问：

```
http://abc.com
```

多次刷新，请求会被轮询分发到 8080、8081、8082 三个端口的应用实例上。

也可以进入 Nginx 容器内测试网络连通性：

```bash
docker exec -it <nginx容器ID> bash
# 容器内执行
curl http:// 192.168.31.250:8080
```

> 注意：Nginx 的基础镜像里没有安装 `ping` 命令，用 `curl` 代替测试连通性。

---

## 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 容器内 `ping abc.com` 报 `command not found` | Nginx 镜像没有内置 `ping` | 改用 `curl` 测试，或 `apt install iputils-ping` |
| 访问 `abc.com` 无响应 | upstream 用了 `localhost` | 改成宿主机局域网 IP（用 `ifconfig` 查） |
| Nginx 启动失败，端口被占用 | 本机 80 端口已有进程 | `lsof -i:80` 查看并关闭占用进程 |
| 修改 nginx.conf 后不生效 | 配置未热更新 | `docker restart <容器ID>` 或 `nginx -s reload` |
| hosts 修改后域名仍解析到公网 | 浏览器 DNS 缓存 | 清除浏览器缓存，或用 `curl abc.com` 测试 |
